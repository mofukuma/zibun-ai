# RAGアプリケーション開発計画書

## 1. 要件（作るもの）

このアプリケーションは、事前に用意した情報ファイルを参考にして、質問に答えてくれるチャットシステムです。

### 主な機能

- テキストファイル（knowledge.txt）に書かれた情報を読み込む
- ユーザーが質問を入力できる
- AIがファイルの内容を参考にして答える
- これまでの会話を覚えていて、続けて話せる
- ブラウザで使える画面がある

### 想定される使い方

1. knowledge.txtに参考情報を書いておく（商品情報、マニュアル、FAQなど）
2. ブラウザでアプリを開く
3. 質問を入力すると、ファイルの内容をもとに回答してくれる
4. 前の会話も覚えているので、「さっきの件について詳しく」などの質問もできる

---

## 2. 使う技術

### メインで使う技術

- **Streamlit（ストリームリット）**: ブラウザで使える画面を簡単に作れるツール
- **LlamaIndex（ラマインデックス）**: ファイルの情報を整理して、AIが参照しやすくするツール
- **Claude API**: Anthropic社が提供する高性能なAI（claude-sonnet-4-20250514モデル）

### その他の技術

- **Python**: プログラミング言語
- **python-dotenv**: 秘密の情報（APIキー）を安全に管理するツール
- **anthropic**: Claude APIを使うためのツール

### なぜこれらを選んだか

- **Streamlit**: プログラムを少し書くだけで、すぐに画面付きアプリが作れる
- **LlamaIndex**: 大きな文書ファイルでも、必要な部分だけを探してAIに渡してくれる
- **Claude API**: 会話が自然で、日本語も得意なAI

## 3. 機能の詳細

### 3.1 ファイル構成

- **main_3.py**: メインのプログラム（すべての機能がこのファイルに入っている）
- **knowledge.txt**: 参考情報を書くファイル（会社情報、製品情報など）
- **requirements.txt**: 必要なツールのリスト
- **.env**: APIキーを保管する設定ファイル
- **.env.example**: 設定ファイルの見本

### 3.2 プログラムの構造（main_3.py）

#### ファイル先頭のコメント

- このプログラムが何をするものか
- 使い方の説明
- 必要な設定

#### 主な機能

**1. 初期設定**

- APIキーを読み込む（.envファイルから）
- 必要なツールを準備する

**2. 知識ファイルの読み込み**

- knowledge.txtファイルを開く
- LlamaIndexを使って、AIが検索しやすい形に整理する
- これにより、大きなファイルでも素早く必要な情報を見つけられる

**3. 画面の表示**

- チャットの画面をブラウザに表示
- タイトルや説明を表示
- これまでの会話履歴を表示

**4. 会話の記憶機能**

- Streamlitのセッション機能を使う
- ユーザーの質問とAIの回答を保存
- ページを更新しない限り、会話が続く
- 「さっき言った〇〇について」のような質問に答えられる

**5. 質問への回答**

- ユーザーが質問を入力
- LlamaIndexが knowledge.txt から関連する情報を探す
- 見つかった情報と、これまでの会話履歴を Claude API に送る
- AIが答えを生成
- 答えを画面に表示し、会話履歴に保存

#### 各関数の役割

**関数1: 初期化処理**

- セッション状態の確認
- 会話履歴の準備

**関数2: 知識ベースの読み込み**

- ファイルの存在確認
- LlamaIndexでインデックス作成
- エラーハンドリング（ファイルが見つからない場合など）

**関数3: 質問処理**

- 知識ベースから関連情報を検索
- 会話履歴と合わせてAIに送信
- 回答の生成

**関数4: メイン処理**

- 画面のレイアウト
- ユーザー入力の受付
- 会話フローの制御

### 3.3 コメントの配置

以下の場所に日本語コメントを入れる：

- ファイルの先頭：プログラム全体の説明
- 各関数の前：その関数が何をするか
- 複雑な処理の前：なぜその処理が必要か、何をしているか

## 4. 開発手順

### ステップ1: 環境準備

1. .envファイルを作成し、APIキーを設定
2. requirements.txtを作成
3. 必要なツールをインストール（`pip install -r requirements.txt`）

### ステップ2: 知識ファイル作成

1. knowledge.txtを作成
2. 参考情報を書き込む（テスト用の内容でOK）

### ステップ3: プログラム作成

1. main_3.pyを作成
2. 各機能を順番に実装
3. コメントを丁寧に書く

### ステップ4: 動作確認

1. `streamlit run main_3.py` でアプリを起動
2. ブラウザで動作確認
3. 質問と回答が正しく動くかテスト

## 5. requirements.txtの内容

```
streamlit
anthropic
python-dotenv
llama-index
llama-index-llms-anthropic
llama-index-embeddings-huggingface
```

各ツールの説明：

- **streamlit**: 画面を作るツール
- **anthropic**: Claude APIを使うツール
- **python-dotenv**: .envファイルからAPIキーを読むツール
- **llama-index**: 知識ベースを作るメインツール
- **llama-index-llms-anthropic**: LlamaIndexでClaudeを使うためのツール
- **llama-index-embeddings-huggingface**: 文章の意味を数値化するツール（類似検索に必要）

## 補足: RAGとは

RAG（Retrieval-Augmented Generation）とは、「検索で情報を集めて、それを参考にAIが答えを生成する」仕組みのことです。

### 通常のAI

知識が学習時のデータのみに限られる

### RAG方式のAI

1. 質問を受け取る
2. 関連する情報をファイルから検索
3. 検索した情報をもとに回答を生成

→ 最新情報や専門的な情報にも対応できる
