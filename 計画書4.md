# 太宰治作品を学習したチャットアプリの計画書

## 1. 要件（このアプリで実現したいこと）

### 基本的な目的

- 太宰治の作品について質問できるチャットアプリを作る
- 青空文庫から自動で太宰治の作品を集める
- 集めた作品の内容をもとに、AIが質問に答える
- 過去の会話を覚えていて、続きの会話ができる

### 具体的な動き

1. アプリを起動すると、自動的に青空文庫から太宰治の作品を取得する
2. 取得した作品をAIが理解できる形に変換して保存する
3. ユーザーが質問を入力すると、関連する作品の部分を探し出す
4. その部分を参考にして、AIが答えを生成する
5. 会話の履歴を覚えているので、「それって何？」のような質問もできる

## 2. 使う技術（どのツールを使うか）

### メインのツール

- **Streamlit（ストリームリット）**
- 役割：ウェブブラウザで動くチャット画面を作るためのツール
- 選んだ理由：少ないコードで綺麗なチャット画面が作れる

- **LlamaIndex（ラマインデックス）**
- 役割：文章を検索しやすい形に整理して、関連する部分を見つけるツール
- 選んだ理由：たくさんの文章の中から必要な部分だけを効率的に探せる

### AIとモデル

- **Claude AI（クロード）**
- モデル名：claude-sonnet-4-20250514
- 役割：質問に対して自然な日本語で答えを生成する
- 選んだ理由：日本語の理解力が高く、長い文章も扱える
- キャラクター設定：太宰治として振る舞い、一人称「私」で自虐的に話す

- **HuggingFace の埋め込みモデル**
- モデル名：intfloat/multilingual-e5-base（日本語対応）
- 役割：文章を数値に変換して、似ている文章を探せるようにする
- 選んだ理由：無料で使えて、日本語の理解力が高い

### データソース

- **青空文庫**
- URL：https://www.aozora.gr.jp/index_pages/person35.html
- 取得方法：ページからリンクをたどって、作品本文だけを自動抽出
- 最大取得作品数：280作品
- 前処理：ルビ（ふりがな）、注釈、ヘッダー、フッターを自動除去

### セキュリティ

- **APIキーの管理**
- .envファイルに保存（パスワードのような重要情報を別ファイルで管理）
- Gitにアップロードしないように設定

## 3. 機能（何ができるか）

### 機能1：作品データの自動収集

- 青空文庫のページから太宰治の作品リストを取得（最大280作品）
- 各作品の図書カードページにアクセスして作品名を取得
- ZIPファイルをダウンロードして本文テキストを抽出
- 不要な部分（ヘッダー、フッター、ルビ、注釈）を自動除去
- 作品名をテキストの先頭に追加（【作品名: ○○】形式）
- 取得したデータをローカルに保存
- プログレスバーで進捗を表示
- サーバー負荷軽減のため0.5秒間隔でアクセス

### 機能2：データの検索準備

- 取得した作品を小さな塊（チャンク）に分割
- チャンクサイズ：1000文字
- オーバーラップ：200文字
- 各塊を数値データ（ベクトル）に変換
- 検索しやすいようにインデックス（索引）を作成
- インデックスをファイルに保存（次回起動時に再利用）

### 機能3：チャット画面

- ブラウザで開けるシンプルなチャット画面
- ユーザーが質問を入力できる入力欄
- AIの回答が表示されるメッセージエリア
- 過去の会話履歴が見られるサイドバー

### 機能4：質問応答

- ユーザーの質問を受け取る
- 質問に関連する作品の部分をベクトル検索（上位5件）
- 検索結果をコンソールに表示（デバッグ用）
- 見つかった部分とユーザーの質問をAIに送信
- AIが太宰治のペルソナで回答を生成
- ストリーミング形式で回答を逐次表示

### 機能5：会話の記憶

- 過去の会話履歴を保存
- 新しい質問をする時に、過去の会話も参考にする
- 「それ」「あれ」などの指示語も理解できる
- 会話をリセットするボタン

## 5. 開発の流れ

### ステップ1：環境準備

1. requirements.txtを作成
2. 必要なライブラリをインストール
3. .envファイルにAPIキーを設定

### ステップ2：データ収集機能の実装

1. 青空文庫のページからリンクを取得
2. 各作品ページから本文を抽出
3. テキストをファイルに保存

### ステップ3：検索機能の実装

1. LlamaIndexの設定
2. テキストの分割とインデックス作成
3. インデックスの保存と読み込み

### ステップ4：チャット機能の実装

1. Streamlitでチャット画面を作成
2. メッセージの送受信
3. 会話履歴の管理

### ステップ5：AI応答機能の統合

1. 質問から関連テキストをベクトル検索（top_k=3）
2. 検索結果を手動でコンテキストとして構築
3. 太宰治のペルソナを設定したシステムプロンプトを作成
4. Claude APIで回答をストリーミング生成
5. 会話の記憶機能を統合

### ステップ6：テストと改善

1. 各機能が正しく動作するか確認
2. エラーハンドリングの追加
3. ユーザー体験の改善

## 6. コーディング規約

### コメントのルール

- **ファイル先頭**：プログラム全体の目的を説明
- **関数の前**：その関数が何をするかを説明
- **複雑な処理の前**：処理の内容をステップごとに説明
- すべて日本語で記述

### コードの書き方

- 変数名は英語でも意味が分かるものを使う
- 1つの関数は1つの役割だけを持つ
- エラーが起きた時の対処も考えて書く

## 7. 注意事項

### セキュリティ

- APIキーは絶対に.envファイルで管理
- .gitignoreに.envを追加してGitにアップロードしない

### パフォーマンス

- インデックスは一度作ったら保存して再利用
- データ取得は初回のみ実行（すでにあれば再取得しない）

### エラー対応

- ネットワークエラーに対応（リトライ機能）
- データ取得失敗時のエラーメッセージ
- APIキーが無い時の警告

## 8. 必要な環境変数（.envファイルの内容）

```
ANTHROPIC_API_KEY=your_claude_api_key_here
```

## 9. 完成後の使い方

1. ターミナルで `pip install -r requirements.txt` を実行
2. .envファイルにAPIキーを記入
3. `streamlit run main_4.py` でアプリを起動
4. ブラウザでチャット画面が開く
5. 初回は自動的に青空文庫からデータを取得（数分かかる）

- プログレスバーで進捗を確認できる
- 取得した作品数が画面に表示される

6. インデックス作成が完了後、太宰治として振る舞うAIと会話できる
