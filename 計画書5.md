# コメダ珈琲店情報RAGアプリケーション開発計画書

## 1. 要件

### 1.1 概要
コメダ珈琲店の東京都内の店舗情報をもとに、ユーザーの質問に答えるチャットアプリケーションを作成します。

### 1.2 機能要件
- **チャット機能**: ユーザーが店舗について質問できる会話型インターフェース
- **店舗情報検索**: エリア、店名、設備(喫煙/電源/WiFi)に関する質問に回答
- **会話記憶**: 過去のやり取りを覚えて、文脈を考慮した回答ができる
- **日本語対応**: 完全に日本語での質問・回答に対応

### 1.3 データ要件
- **データソース**: `komeda_list_tokyo.csv`
  - エリア情報
  - 店舗名
  - 喫煙の有無
  - 電源の有無
  - WiFiの有無

## 2. 使用する技術

### 2.1 フレームワーク・ライブラリ

#### **Streamlit**
- Webアプリケーションの画面を作成するツール
- チャット画面を簡単に作れる
- ブラウザで動作するので、特別なインストールが不要

#### **LlamaIndex**
- RAG(情報検索拡張生成)を実現するためのツール
- CSVデータを検索可能な形式に変換
- ユーザーの質問に関連する情報を自動的に見つける

### 2.2 AI・モデル

#### **Claude Sonnet 4**
- モデル名: `claude-sonnet-4-20250514`
- Anthropic社が提供する高性能な言語モデル
- 自然な日本語での会話が可能
- 質問に対して正確で詳しい回答を生成

#### **埋め込みモデル**
- モデル名: `intfloat/multilingual-e5-base`
- HuggingFaceで提供される多言語対応の埋め込みモデル
- 日本語テキストを数値データに変換
- 質問と店舗情報の関連性を計算するために使用

### 2.3 環境管理

#### **.env ファイル**
- APIキーなどの機密情報を安全に管理
- `ANTHROPIC_API_KEY`を保存

#### **requirements.txt**
- 必要なPythonパッケージのリスト
- プロジェクトの依存関係を明確化

## 3. 機能設計

### 3.1 データ準備機能

#### **CSVファイルの読み込み**
- `komeda_list_tokyo.csv`を読み込む
- 各行のデータを構造化された情報として扱う

#### **テキスト形式への変換**
- CSVの各行を以下の形式の文章に変換:
```
【エリア】新宿区・渋谷区・豊島区
【店名】コメダ珈琲店 池袋西武前店
【喫煙】なし
【電源】あり
【WiFi】あり
```

**なぜこの形式にするか**:
- 各行に項目名を含めることで、データの一部だけを取り出しても意味が分かる
- AIが情報を理解しやすくなる
- 検索精度が向上する

### 3.2 RAG (情報検索拡張生成) 機能

#### **インデックス作成**
1. 変換したテキストを埋め込みモデルで数値データ化
2. 検索可能なインデックスを作成
3. アプリ起動時に一度だけ実行

#### **検索機能**
1. ユーザーの質問を受け取る
2. 質問に関連する店舗情報をインデックスから検索
3. 見つかった情報をClaudeに渡す

### 3.3 チャット機能

#### **会話インターフェース**
- Streamlitのチャット機能を使用
- ユーザーメッセージを画面右側に表示
- AIの回答を画面左側に表示

#### **会話記憶機能**
- セッション内の全ての会話を保存
- 過去の質問と回答を参照可能
- 「さっき言った〜」などの表現に対応

**実装方法**:
```python
# Streamlitのセッションステートに会話履歴を保存
if "messages" not in st.session_state:
    st.session_state.messages = []
```

### 3.4 AI応答生成

#### **Claude APIの呼び出し**
1. ユーザーの質問と検索された店舗情報を組み合わせる
2. Claude APIにリクエストを送信
3. 生成された回答をチャット画面に表示

#### **回答の品質向上**
- 検索した店舗情報を必ず参照するよう指示
- 存在しない情報は推測しないよう指示
- 丁寧で分かりやすい日本語で回答するよう指示

## 7. 技術的なポイント

### 7.1 RAGの仕組み

#### **従来のAI**
- AIの知識だけで回答
- 最新情報や特定のデータには対応できない

#### **RAG (このアプリ)**
1. ユーザーが質問する
2. 質問に関連する店舗情報を検索
3. 検索した情報をAIに渡す
4. AIが情報を基に正確な回答を生成

**メリット**:
- 正確な店舗情報を提供できる
- データを更新すれば常に最新情報に対応
- AIが「知らない」と言わず、データを参照して回答

### 7.2 埋め込みモデルの役割

#### **テキストの数値化**
```
「池袋の店舗」 → [0.2, 0.8, -0.3, ...] (ベクトル)
「コメダ珈琲店 池袋西武前店」 → [0.3, 0.7, -0.2, ...] (ベクトル)
```

#### **類似度計算**
- ベクトル同士の距離を計算
- 距離が近い = 意味が似ている
- 質問に最も関連する店舗情報を見つける

### 7.3 会話記憶の実装

#### **Streamlitのセッションステート**
```python
# 初回アクセス時
st.session_state.messages = []

# メッセージ追加
st.session_state.messages.append({
    "role": "user",
    "content": "池袋の店は?"
})

st.session_state.messages.append({
    "role": "assistant",
    "content": "池袋には2店舗あります..."
})
```

- ブラウザを閉じるまで会話履歴を保持
- リロードしても会話が消えない
- 全ての過去の質問・回答を参照可能
